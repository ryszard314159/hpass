Settings

Basic Usage

  1. Enter password hint in the Enter Hint box.
  2. Generate password by pressing Enter key or clicking on an rigth arrow button in the top-upper corner.
  3. Paste password from the clipboard where you need it.
Important: generated password is uniquely defined by a few simple settings, which are hidden by default to prevent shoulder surfing. To access, and potentially change, these settings, click button in the top-left corner. If you plan to use this tool on multiple devices remember that you need to have the same settings on each device.

Read on to find out more...

Installation and Settings

When you first visited the HPASS site, settings were initialized and saved locally on your device. You migh have noticed that the Secret was initialized to some random alphanumeric string. This ensures that the passwords generated for each user are unique even if all other parameters remain the same.

To modify and save setting: (1) enter the settings you want (2) clear Enter Hint box and (3) generate password.

Press Reset button to populate generic settings fields with default values:

  • Secret (ü§´): "Replace Me!" - you should change it to some value you are likely to remember.
  • Special Character (🌶): "_" i.e. underscore (multiple characters are allowed).
  • Length (üìè): 15
Special Character and Length values were selected to satisfy the requirements for the majority of sites. You can change these values if you need to. You are strongly encouraged either to keep Secret generated for you at the begining or to change to some personalized value. When password is generated with non-empty Enter Hint box and any of these three values is different from the value in generic settings these values will be stored in site-specific settings on your local device and used automatically next time you enter the same string in the Enter Hint box. This is useful when you encounter a site which does not accept Special Character and/or Length values from your generic settings.

Multiple devices

You need to have the same settings on each device to generate the same passwords. To make this synchronization easier, site-specific settings can be exported () from one device and imported () on another using simple JSON file format. If some site does not accept either Special Character or Length in your generic settings you have to enter values acceptable for the site in these boxes and generate the password. These modified values will be saved locally in site-specific settings.

As an example, let's assume that abc site does accept only these !#$% four special characters and xyz accepts only 6-12 characters long passwwords. If you export your site-specific settings after generating acceptable passwords for these two sites the file you export might look like this: {"xyz": {"length": 12}, "abc": {"pepper": "!"}} - assuming you used "abc" and "xyz" in Enter Hint box to generate passwords.

Extra security

The purpose of two additional settings is to provide extra security layer in case your device is compromised and possibly your Secret stolen.

These variables:

  • Top Secret (🤐): any string you wish (default: empty string)
  • Burn (🔥): number, in 0-999999 range, of warm up rounds for Pseudo-Random Number Generator (PRNG) to use (default: 0)
are never stored anywhere, and discarted at the end of the session, or when the page is reloaded. After they are discarted they need to be entered again if you want to use them to generate password. This is a small price in convenience you may want to pay for high value passwords (bank accounts, social security, etc.) if you are concerned about possibility of data on your device being compromised.

Password prompt (or hint) entered in the Enter Hint box at the top of the screen, and Secret, Special Character Length, Top Secret are used to construct the seed for PRNG. This seed, together with Burn parameter (number of warm up rounds for PRNG), determines uniquely the generated password i.e. with the same values always the same password will be generated.

NOTE: it is important to store Secret (and other values if different then default settings) in safe place in case you loose data on your device (or device itself). In such a case, you would need to enter these settings again in order to recreate your passwords - remember, these values (by design) are not stored in any central location, only on your device, and cannot be recovered if you loose them - Top Secret and Burn are never stored.

Usage

  1. Enter password hint in the Enter Hint box. You can use any string as a hint, but it is probably the easiest and most straighforward to use a secondary domain of the site for which you need a password e.g. abc for https://www.abc.com. For convenience you can simply copy and paste https://www.abc.com and abc will be automaticall extracted and used as a hint.
  2. For extra security you may want to enter custom values for Top Secret and Burn. These parameters are never stored, so in case your device is compromised and bad actors manage to steal your data this offers an extra security layer.
  3. Generate password and copy it to the clipboard.
  4. Paste generated password from the clipboard where you need it.
