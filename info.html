What is HPASS?

HPASS(™) is a lightweight, intuitive, simple and easy to use tool to create strong passwords.

Why?

If you think there is a way for improvement in your current approach to password handling you may find HPASS helpful. One way to store and share passwords is by using a password manager. It is an often recommended and widely used solution, but not without problems.

HPASS(™) uses an alternative approach with one major advantage: passwords are generated locally, on demand and never stored in any central location. Passwords and few simple settings to generate them (Secret, Special character and Length) are never stored in any centralized location, only on your local device, so there is no danger of data breach so often reported for password managers.

Large number of sites and apps offer strong random password generation, but quite extensive search (see links below) failed to find any which would allow to generate reproducible, hint-derived passwords as HPASS does.

Typical usage

Assume you have HPASS installed and need a password for https://www.abc.com website. You simply enter abc as a password hint into enter hint input box and press generate button () This will generate password string (57MDf_e1QkNIrD8 assuming Secret = "SECRET", and default values for Special character and Length) and copy it to the clipboard. After that you can paste it where it is needed. As a convenience feature, if you enter the whole URL (e.g. https://www.abc.com) the app will atomatically extract abc (secondary domain) as a password hint.

Note that the default settings (Special character = "_", Length = 15) have been selected to satisfy most (if not all) websites password requirements - some websites will not accept e.g. ? as symbol, and some impose rather short maximum password length - 15 seems to be accepted by most.

Please note that the way HPASS generates password from given input parameters (hint, Secret, Special character, Length) is one-way hash function and the input parameters cannot be recovered from the password string.

Security considerations

There is always a balance between security and convenience. In HPASS context it means that although keeping only local copies of any sensitive information is more secure and less tempting for any hackers then keeping it in a centralized location - just compare relative effort and potential payoff of braking once into central database with millions of password vs braking into millions of individual devices - no information is 100% secure if it is written down, the only secure data is the one that is kept in your head (barring any mind-reading :). If you are concerned about the possibility of some bad actors braking into your device and stealing settings you use to generate password (Secret, Special Character and Length), there are some steps you can take to vastly improve security of your password by paying (small) inconvenience price. For example you can:
  1. "decorate" hint with some extra characters e.g. instead of using abc you can always append (or prepend) some character to it
  2. use generated password as a hint and generate password again
...and probably you can invent many more simple schemes to accomplish the same goal i.e. that the generated password cannot be derived from any stored data, since the part of the whole password generating procedure is stored only in your mind :) Of course, if the sites you need to login into where the security is most important (bank accounts, etc) offer multifactor authentication (MFA) it is always a good idea to use it.

Last, but not least, all HPASS code is open source - you can inspect it, comment on it, propose and contribute improvements.

Problems with password managers

You can find multiple sources discussing pros and cons of using password manager. Very short list is:

  • Pros: convenience, ability to share passwords
  • Cons: need to remember master password, vulnerability to hacking, safety

We can argue that HPASS has all adantages and none of dissadventages of password managers.
