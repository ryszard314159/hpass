What is HPASS?

HPASS(™) is a lightweight, intuitive, simple and easy-to-use tool to create strong passwords.

Why?

If you think there is room for improvement in your current approach to password handling you may find HPASS helpful. One way to store and share passwords is by using a password manager. It is an often recommended and widely used solution, but not without problems.

HPASS(™) uses an alternative approach with one major advantage: passwords are generated locally, on-demand and never stored in any central location. Passwords and few parameters to generate them ( Hint, Secret, Special Character, and Length) are never stored in any centralized location, only on your local device, so there is no danger of data breach so often reported for password managers.

Large number of sites and apps offer strong random password generation, but quite extensive search (see links below) failed to find any which would allow to generate reproducible, hint-derived passwords as HPASS does.

UPDATE [2024-03-01]: Apparently Master Password (rebranded now as Spectre https://spectre.app) was developed in 2012. It uses basically the same idea as HPASS - do not store password, generate reproducible password when you need it. You can find there extensive explanations why this is a better solution then storing passwords, as password managers do. However, the whole process Spectre uses seems overcomplicated and restrictive at the same time. For example, in addition to the HPASS Secret eqivalent,it requires user name, it generates passwords only following few fixed templates, etc. Take a look, and decide for yourself.

Typical usage

Assume you need a password for a website: URL=https://www.abc.com. You can simply enter abc as a password hint into "Enter Hint" input box and press generate button (). This will generate password string and copy it to the clipboard, and you can paste it where it is needed. As a convenience feature, if you enter the whole URL the app will automatically extract secondary domain (i.e. "abc" in this example) as a password Hint.

Notes

  • The default settings (Special Character = "_", Length = 15) have been selected to satisfy most password requirements. Some websites will not accept certain special characters, and some impose rather short maximum password length - "_" and 15 seems to be accepted by most (if not all) sites.
  • Purpose of Secret is to create different passwords for users using the same other parameters.
  • HPASS generates password from given input parameters through one-way hash function and the input parameters cannot be recovered from the password string.

Security considerations

Keeping only local copy of any sensitive information is more secure and less tempting for any potential hackers then keeping it in a centralized location - just compare relative effort and payoff of breaking once into central database with millions of password vs breaking into millions of individual devices. However, no information is 100% secure if it is written down, and the only secure data is the one that is kept in your head (barring any mind-reading capabilities :). Given that, if you are still concerned about some bad actors breaking into your device and stealing settings you use to generate password, there are some steps you can take to make your passwords virtually 100% secure by paying (small) inconvenience price. For example you can:
  1. Use To Secret and Burn for extra layer of security. These two parameters are never stored.
  2. "decorate" hint with some extra characters e.g. instead of using abc you can always append some character(s) (known only to you!) to it
  3. use generated password as a hint and generate password again
You can probably invent more simple schemes to accomplish the same goal i.e. that the part of the whole password generating procedure is stored only in your head :) Of course, if the sites you need to login into where the security is most important (bank accounts, etc.) offer multifactor authentication (MFA) it is always a good idea to use it.

Last, but not least, all HPASS code is open source - you can inspect it, comment on it, propose and contribute improvements.

Problems with password managers

You can find multiple sources discussing pros and cons of using password manager. Very short list is:

  • Pros: convenience, ability to share passwords
  • Cons: need to remember master password, vulnerability to hacking, safety, lock-in for browser build-in password managers.
